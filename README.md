# Emergency Resource Sharing under Scenario-Switching Distributionally Robust Optimization

This repository provides an implementation of a scenario-driven framework for emergency resource sharing based on a Distributionally Robust Optimization (DRO) model. The framework is designed for emergency resource coordination under major public health emergencies, where disaster scenarios evolve over time and emergency actors, responsible for specific tasks, need to share limited resources efficiently.

The core workflow includes overall demand-level identification, scenario-based demand generation, and resource sharing optimization. The repository also provides a series of experiments, including out-of-sample evaluation, initial scenario analysis, external support analysis, scenario evolution experiments, comparison with SAA, Wasserstein radius sensitivity analysis, parameter analysis, and computational complexity analysis.

## Dependencies and Installation
- Python >=3.9 
- Packages: numpy, pandas, gurobipy, cvxpy.
- Gurobi (required for MILP solver)

It is recommended to create a dedicated environment:
```
conda create -n dro-gurobi python=3.9 -y
conda activate dro-gurobi
conda install -c gurobi gurobi
```

## Overall Demand-Level Identification

The BWM-TOPSIS procedure is used to identify the overall demand level for six unit types (C1–C6) under each scenario. The results are used as inputs for subsequent demand sample generation and optimization experiments.

The scoring matrices and optimal factor weights are stored in:
- BWM_Topsis/Score_Weight/

The normalized scoring matrices are stored in:
- BWM_Topsis/Normalized_matrix_data/


## Demand Sample Generation

Demand samples are generated by:
```
python Genaration_Sample/generate_sample_data.py
```
- Output: Sample_data/

## Resource Sharing Optimization Experiments

1. **Out-of-Sample Experiment**  
```
python Experiments/post_initial_scenario.py
```
- Output: Experiments/result/

2. **Initial Scenario States Experiment**  
```
python Experiments/post_initial_scenario.py
```
3. **External Support Experiment**  
```
python Experiments/external_support.py
```
4. **Scenario Evolution Experiment**  
```
Experiments/Scenario_evolution/evolution_s2/
Experiments/Scenario_evolution/evolution_s7/
Experiments/Scenario_evolution/evolution_s13/
```

5. **Comparison with SAA**  
```
python Experiments/SAA_model.py
```

6. **Wasserstein Radius Experiment**  
```
python Experiments/radius.py
```

7. **Parameter Experiment**  
```
python Experiments/parameters.py
```

8. **Computational Complexity Experiment**  
The computational complexity experiment evaluates the scalability of the proposed model under different combinations of the number of emergency actors N and resource types R. The experiment files are stored in:
- Experiments/Computational_complexity/

Each subfolder corresponds to a specific combination of N and R, such as:
```
18_6/
22_6/
36_9/
50_12/
92_12/
```

### General Running Workflow
```
# Step 1: Solve optimization model
python Experiments/post_initial_scenario.py

# Step 2: Calculate sharing quantities
python Experiments/sharing_number.py

# Step 3: Evaluate performance
python Experiments/evaluation.py
```
